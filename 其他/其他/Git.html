# Git

# 安装

安装之后需要输入以下代码

$ git config --global user.name "Your Name"
$ git config --global user.email "email@example.com"

# 创建版本库

版本库：别名仓库，英文名 repository

  1. 创建目录
$ mkdir mygit
$ cd mygit
$ pwd

pwd：显示当前目录

  1. 通过git init将这个目录变成 Git 可以管理的仓库
$ git init

如果你没有看到.git目录，那是因为这个目录默认是隐藏的，用ls -ah命令就可以看见。

# 添加文件Git

# vim 编辑器常用操作

用 git 命令行提交文件时，默认使用 vim 编辑器，基本操作：

  • 按 a ,i 或o进入编辑模式
  • 按esc退出编辑模式
  • 在操作模式下，:wq为写入退出，:q!不保存退出

# 编写文件并提交

编写gu.txt文件

文件一定要放到mygit目录下（子目录也行），因为这是一个Git仓库，放到其他地方Git再厉害也找不到这个文件。

添加文件到Git仓库，分两步：

  1. 使用命令git add gu.txt，注意，可反复多次使用，添加多个文件；
  2. 使用命令git commit -m "wrote a file"，完成。

# 修改文件并查看状态和改动

修改gu.txt文件

git status： 查看仓库当前的状态，gu.txt虽然被修改过，但是还没有被提交 git diff：diff也就是difference，查看做出的改动

# 修改文件后提交文件

  1. 添加要提交的文件
$ git add gu.txt
  1. 提交前，查看状态
$ git status
  1. 提交文件
$ git commit -m "edit"

# 查看历史记录并返回特定版本

git log： 时间排序显示提交的日志 git log --pretty=oneline：简化日志信息 git reset --hard HEAD^：返回上一个版本 git reflog：查看命令历史，已确定要回到哪个版本 git reset --hard [版本号，不一定是要全部，记得前几位即可，记不得就git reflog]：重返特定版本

HEAD就是一个指针，指向想还原的版本，而版本可以建立许多分支

# 名词解释

  1. 工作区（Working Directory）：比如mygit就是一个工作区
  2. 版本库（Repository）：工作区里面有一个隐藏的文件夹.git,其中index（或者叫stage）被称为暂存区，

# 不同场景撤销操作

  1. 修改了文件，但是还没提交到暂存区，git checkout -- file
  2. 修改了文件，但是提交到了暂存区
    • git reset HEAD -- file
    • git checkout -- file
  3. 修改了文件，并提交到了版本库
    • git reset --hard HEAD^

# 删除文件

git rm file:提交删除文件

# 提交到远程仓库

第1步：创建SSH Key。在用户主目录下，看看有没有.ssh目录，如果有，再看看这个目录下有没有id_rsa和id_rsa.pub这两个文件，如果已经有了，可直接跳到下一步。如果没有，打开Shell（Windows下打开Git Bash），创建SSH Key：

$ ssh-keygen -t rsa -C "youremail@example.com"

id_rsa是私钥，不能泄露出去，id_rsa.pub是公钥，可以放心地告诉任何人。

第2步：登陆GitHub，打开“Account settings”，“SSH Keys”页面：

然后，点“Add SSH Key”，填上任意Title，在Key文本框里粘贴id_rsa.pub文件的内容

$ git remote add origin git@github.com:michaelliao/learngit.git

$ git push -u origin master

由于远程库是空的，我们第一次推送master分支时，加上了-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。

从现在起，只要本地作了提交，就可以通过命令：

$ git push origin master
